# Security Policy

## Reporting Security Vulnerabilities

I take security issues seriously. If you discover a security vulnerability, please follow these guidelines:

### **Do NOT** create a public GitHub issue for security vulnerabilities.

Instead, please report security issues privately by:
- Joining our Discord server: https://discord.gg/uZsCCdPyF3
- Contacting the project maintainer via Discord DM

## What to Include in Your Report

When reporting a security vulnerability, please provide:
- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact assessment
- Suggested fix (if you have one)
- Your contact information for follow-up

## Response Timeline

- **Fix Timeline**: Varies based on complexity and severity

## Security Best Practices

When using RAC Clicker:
- Only download releases from official GitHub releases page
- Verify file integrity before running
- Run the application with minimum required privileges
- Keep your system and dependencies updated

## Scope

This security policy applies to:
- RAC Clicker application code
- Build scripts and release processes
- Dependencies with known vulnerabilities

**Note**: This policy does not cover third-party servers or services that users may interact with while using the application.

## Acknowledgments

We appreciate responsible disclosure and will acknowledge security researchers who help improve the security of RAC Clicker.
